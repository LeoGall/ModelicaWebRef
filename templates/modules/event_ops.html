Event Operators
pre {{ example }}
Usage Description
pre(y) At an event, the pre operator is used to refer to the value of a variable prior to that event. There are several caveats related to the use of pre. These can be found in {{ xref("3.7.3", 29) }}.

noEvent {{ example }}
Usage Description
noEvent(expr) The noEvent operator suppresses the generation of events by any event generating operators or functions in the expression expr. Further information about the noEvent operator can be found in {{ xref("3.7.3.2", 29) }} and {{ xref("8.5", 88) }}.

change {{ example }}
Usage Description
change(v) The expression change(v) because true only for the instant that the value of the variable v changes (i.e., v<>pre(v)). See the pre operator and/or {{ xref("3.7.3.1", 30) }} for more details and restrictions.

edge {{ example }}
Usage Description
edge(b) The edge operator becomes true only at the instant when the expression b goes from false to true (i.e., b and not pre(b)). See the pre operator and/or {{ xref("3.7.3.1", 30) }} for more details and restrictions.

initial {{ example }}
Usage Description
initial() The initial operator is true during the initialization phase of the simulation.

terminal {{ example }}
Usage Description
terminal() The terminal operator becomes true at the end of a successful simulation.

reinit {{ example }}
Usage Description
reinit(x, expr) The reinit operator sets the value of the variable x to be equal to the of expr. Note, reinit can only be used within a {{kw("when")}} clause. For more details on the usage and restrictions of reinit, see {{ xref("8.3.6", 86) }}.

sample {{ example }}
Usage Description
sample(start, interval) The sample operator because true when time>=start and again every interval seconds after that. Further details can be found in {{ xref("3.7.3", 29) }}.

smooth {{ example }}
Usage Description
smooth(p, expr) When evaluated, the smooth operator simply returns the value of the expression expr. But it also acts as a guarantee that expr is p times continuously differentiable. For further details, see {{ xref("3.7.3.2", 30) }}.

terminate {{ example }}
Usage Description
terminate() The terminate operator instructs the underlying simulation environment to terminate the simulation after the completion of the next integration step or event iteration. See {{ xref("8.3.8", 87) }} for more details.
